## Авто-сборщик PluginManager под Linux
Скрипт стандартизирует процесс вокруг трёх путей:

- `build/_staging/refs` - временная папка для общих DLL, нужных во время компиляции.
- `build/Mods/1_PluginManager` - финальная папка мода.
- `build/Mods/1_PluginManager/Plugins/<PluginName>` - финальная папка конкретного плагина.

Локальные копии игровых DLL в `libs` каждого проекта больше не нужны. Скрипт берёт ссылки напрямую из установленного сервера:

- `<server-root>/7DaysToDieServer_Data/Managed`
- `<server-root>/Mods/0_TFP_Harmony`

Для Linux-сборки через `dotnet msbuild` скрипт автоматически использует `/usr/lib/mono/4.8-api`, если эта папка существует. Это даёт reference assemblies для `.NET Framework 4.8`.

## Как Добавить Плагин
Плагины обнаруживаются автоматически по шаблону `<папка>/src/<Имя>/<Имя>.csproj` в корне бандла. Достаточно положить папку плагина рядом с остальными:

- имя `.csproj` = имя DLL (`Plugins/<Имя>.dll`) и идентификатор плагина, по которому `PluginManager.Core` резолвит конфиг и локализацию;
- `static/config.json` копируется в `Config/<Имя>.json`;
- `static/lang/<culture>.json` (или `lang/` в корне плагина) копируется в `Lang/<Имя>.<culture>.json`.

`build.sh` соберёт DLL в `Plugins/`, а сторонние зависимости — в корень мода. Список плагинов не ведётся: `PluginManager.Core` при старте грузит все `Plugins/*.dll` по алфавиту. Чтобы отключить плагин, перенеси его DLL в `Plugins/disabled/` — эта папка не сканируется.

## Интерактивная Сборка
`./build.sh` показывает нумерованный список найденных плагинов:

- Enter — полная пересборка (чистит `build/`, собирает библиотеки, ядро и все плагины);
- номера через пробел — пересборка только выбранных плагинов (ядро переиспользуется, если уже собрано; `--core` форсирует его пересборку);
- `0` — выход.

Без терминала (pipe, cron) меню пропускается и выполняется полная сборка.
